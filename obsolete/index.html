    情人

    明夷虫

    07-17

    多年以前，我和她有过许多次美好的约会。昨天，她路过这个城市，就来看我。

    微黄的路灯下，我们有些拘谨，互相打量，欲语还休。弥散着数十年的往事，浸润着眼角的沧桑。我们一起走了三十分钟的路，我不知是该高兴还是忧伤。

    妻打电话说和孩子在小区花园里的滑梯旁等我。我的孩子今年五岁。我认真地对她说，去见见我的家人。

    孩子在滑梯下玩闹。妻在一旁很开心地笑着，看见了我，就亲昵地跑过来抱住了我。她在一旁看着这些，不说话，只是微笑。

    我们和孩子手拉着手，一起在灰色的夜里奔跑。孩子非常喜欢她，问她可不可以留下来和我们一起生活。她伸出洁白的手轻抚孩子的脸，不说话，只是微笑。

    我知道她会离去，就像许多年前那样。她走了之后，我的世界满是泥泞。

    我对孩子说，她会和我们呆上几天，然后就要去北方，那里的天空有时会绽放美丽的光。

    我向孩子隐瞒了一点现在的真相。她离去之后，世界会继续寒冷和僵硬。这一切，还需要慢慢习惯。

    爱情

    明夷虫

    07-16

    我一贯自以为是，思考过许多与人生观有关的问题，唯独未曾认真思考过爱情。估计是我的爱情太多了，一时半会也思考不出来什么。

    文学作品写爱情，从诗经就开始写，到现在已经两千多年了。人类想明白了么？

    肯定还没有，否则就不会有那么多人还在写爱情小说。不仅还在写，而且内容也逐渐细化了起来，校园爱情，乡村爱情，都市情感，同性爱情，情色抑或色情，甚至还有人兽爱情——

    我看过次数最多的三篇爱情小说，《一个陌生女人的来信》，《茵梦湖》以及《伊豆的舞女》，从这些小说里，只学会了情绪上的忧伤，以为爱情就是一种令人忧伤的东西。这种忧伤来自于，眼前这么多美好的姑娘，得不到她们的芳心，也就罢了……问题是，连身体也得不到啊。

    有时看到那么多的男人在电视剧里抱怨或愤怒，「我得到的只是你的身体，而不是你的心」，就觉得他们很蠢，他们不珍惜自己的女人的身体。当然，也有些女人在电视剧里很勇敢地说，「你只能得到我的身体，得不到我的心」，这时又觉得这些女人也很蠢，因为她们也不珍惜自己的身体。

    不过，看起来，爱情这种东西，与外在无关。

    真的是这样吗？

    不知道。

    只知道，还是要看脸的。不仅要看脸，还要看身材。不仅要看身材，还要看家世。不仅要看家世，还要看工作，看爱好，看星座……要看的东西这么多，还有多少时间去看心呢？再说了，心，是啥，又该如何看？

    还有一些人会有一些奇怪的理论。譬如，女人爱一个男人，就会为他生个孩子，而且会很爱这个孩子。于是，为了爱情，还要再搭上孩子。

    好久没用我的「如无必要，勿设变化」剃刀了。既然要看的东西这么多，是这些东西不存在，还是爱情本身就不存在？

    爱情，需要这么多的祭品的爱情，存在么？

    哲学家很少讨论爱情。他们到现在连人为啥要活着还没搞清楚，没太多时间思考爱情。

    那么科学家呢？他们往往有意无意地暗示，爱情不过是动物传播基因的本能而已，不过是脑子里的多巴胺分泌过多而已。

    社会学家呢？爱情，不过是维护社会安定耍流氓而已，而且美好的爱情也会死，死于婚姻这座坟墓。

    太过于神圣的东西，其命运终归躲不过受践踏，变得一文不值。

    那么，就干脆让这种东西彻底消失吧。让它不存在。这样就可以不献祭，也可以不践踏。大家可以把时间节省下来，去干一些别的无聊的事。这种事，大家都不怎么想做，因为实在太无聊了。所以，我来。

    「如无必要，勿设变化」，这柄剃刀，它的背后其实有着更深刻的原理，那就是，复杂的东西，它的原理必定是简单的，否则它不能维持自身的长久存在。

    爱情如此复杂，又如此长久存在，那么它的原理必定是简单的。

    对爱情最简单的描述，应该是，「窈窕淑女，君子好逑。求之不得，辗转反侧」，这是单恋。或者，「有女怀春，吉士诱之，舒而脱脱，无感我帨，无使尨吠」，这是两情相悦。要研究爱情，须从这种原始描述开始，而不是从现代被一堆乱七八糟的东西囚禁起来的描述开始。

    无论是单恋，还是两情相悦，它们的共同之处在于，所爱之人在自己心目中应当足够美好。这个时候，我没那么多闲工夫去考虑献祭，耍流氓，传播基因，分泌多巴胺，结婚……我爱的是这种美好。只有这种美好是客观存在的，其它任何东西在这种美好存在的时刻，都是不存在的，它们不过是在这美好飘逝之后而强加追忆，本质上都是献祭，只是祭品有所不同。

    这种美好，后来去了哪里？

    哪里都没去，它还在我心里。我甚至没法用文字或其他形式将它抛弃。所以，我很难向任何一个人描述它，更不要说去写爱情小说或情诗了。以至，但凡遇到自己极欣赏的女性，我只能简单地冠之以女神的花环。倘若我心里缺乏这种美好，可能会无法欣赏一切女性。幸好，我的女神很多，她们甚为可爱。只是这种可爱，她们自己无法想象。

    我觉得，爱情，一旦有了，它就会变成生命的一部分。婚姻埋葬的不是爱情，而是我们自己。我们自己也可能不是婚姻埋葬了，而是我们自己闭上了眼睛，然后抱怨天太黑。可能问题始终出在，我们并不像自己所以为的那样了解自己，但是又坚信世上只有自己最了解自己。

    人是无法了解自己的。倘若你真的很相信自己最了解自己，那么从小到大，这个所谓的最了解的自己，不知变了多少次了。倘若你真的相信自己，那你就是个一成不变的人，这是一种病，幼稚病。

    不少人，在爱上一个人的时候，认为这个人是自己的全部，一旦失去，便痛不欲生，从而干出一些难以预料之事。这种行为可视为心灵投射，认为自己所臆想的东西是客观存在的，不存在也要创造出来让它存在。世俗的爱情，需要按照这种套路去演绎，否则写小说的，写剧本的，拍电视/电影的，都不太好混。

    爱情是一种功能，不是一种形态。它可以让我看到一个美好的自己，或者自己美好的一面。认为自己所爱的是一个具象的人，进而要求这个人的所作所为必须对得起自己的爱，这是最大的移情别恋——将对自己的爱，转移到了他人身上。

    爱情这种功能，可以让我美好。无论我爱的人是不是还在我的身边，我都可以保持这种美好。

    那么，两情相悦呢？

    彼此从对方身上看到了自己的美好，而且愿意做对方的镜子。

    爱情，也许就是这么简单。

    Nikola 琐事

    明夷虫

    04-26

    Nikola 是我用了 3 年的静态网页生成工具，这个 Github 主页上的所有页面皆由它生成。它的安装与基本用法可参考《Nikola 入门文档》，对于 Linux 环境，建议在 virtualenv + python 3 环境中安装它。它的更详细的用法，见《Nikola 手册》。本文仅记录我在使用 Nikola 过程中遇到的一些小问题以及不甚高明的对策，仅作备忘之用。

    自定义首页

    Nikola 会自动生成首页，将 10 近期的文章内容收集到一个页面，该页面对应的文件为 index.html。这份文件默认会被放到站点的根目录里，所以，它就成了站点的首页。我觉得 Nikola 这个功能很鸡肋，因为我想自己定义首页，而不是这种自动生成的东西。该怎么办呢？

    首先，在 conf.py 文件里，将 INDEX_PATH 设为

    INDEX_PATH = "blog"

    这样，Nikola 就会将这个自动生成的 index.html 文件放到 站点根目录/blog 目录里，从而令它失去了担当站点首页的资格。理论上，也可以随便取个目录名来用，例如 ·

    INDEX_PATH = "obsolete"

    用这种乾坤大挪移的办法，算是将这个鸡肋的站点首页「消灭」了。接下来，就是建立一个自定义的首页，以后手工来维护它。方法很简单，只需要在 PAGES 目录，任意建立一份源文件，将其 slug 设为 index.html 即可。

    PAGES 目录，就是在 conf.py 文件中设定的目录。例如，我的设定是：

    PAGES = (
    ("stories/*.md", "", "story.tmpl"),
)

    这就意味着，我可以在 stories 目录建立 slug 设为 index.html 的源文件（至于它对应的文件名是什么，并不重要）。假设这份源文件是 Markdown 格式，那么所谓 slug 就是文件头中的 slug 条目，例如：

    <!-- 
.. title: 
.. slug: index
-->

    剩下的内容可自由定义。

    当网站内容重新发布时，这份源文件就会被转化为网站根目录中的 index.html 文件。因为 Nikola 就是这样对待 PAGES 目录中的源文件的，会把它们「平铺」在网站的根目录里。
